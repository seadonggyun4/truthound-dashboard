# 3.0 Upgrade

## Breaking changes

- The dashboard now targets `truthound>=3.0.0`.
- Drift comparison uses `truthound.drift.compare`, not `th.compare`.
- Checkpoint integrations use `CheckpointResult.validation_run` and
  `CheckpointResult.validation_view`.
- Removed product areas no longer ship routes or API entrypoints.

## Surface reset

- Validation, profiling, drift, privacy, checkpoints, reports, plugins,
  lineage, observability, storage, anomaly, and enterprise-sampling remain.
- Dashboard-only mock and compatibility layers are removed.

## Migration guidance

- Migrate only source metadata, validation history, and checkpoint-oriented
  scheduling metadata that directly map to the 3.0 surface.
- Rebuild deleted feature workflows against supported Truthound namespaces
  instead of preserving legacy dashboard-only behavior.
