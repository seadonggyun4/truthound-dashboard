# 3.0 Upgrade

Truthound Dashboard 3.0 is a product reset, not a thin compatibility layer over older
dashboard behavior. The upgrade aligns the dashboard with Truthound 3.0 and removes
surfaces that duplicated engine behavior or represented abandoned product directions.

The most important changes are structural.

- Truthound 3.0 is now the single source of truth for validation, profiling, drift,
  privacy, checkpoint, lineage, reporting, and plugin runtime behavior.
- The dashboard acts as the operational control-plane for sessions, RBAC, secrets,
  ownership, incidents, queues, saved views, artifacts, and observability.
- Reporting REST contracts moved to `/artifacts`. The `/reports` browser route remains
  only as a UI alias for the artifact index.
- Secret-bearing configuration moved to `secret_refs`.
- Saved view scope is explicitly limited to `sources`, `alerts`, `artifacts`, and
  `history`.
- Legacy product areas such as catalog, collaboration, watcher-style schema products,
  mock surfaces, and old report-history APIs were removed.

Operators upgrading to 3.0 should review four areas before treating the deployment as
finished.

1. **Permissions**: confirm roles now resolve through normalized permissions.
2. **Artifacts**: update any automation or documentation that previously referenced
   report-only REST paths.
3. **Secrets**: move operational practice to create and rotate flows instead of reading
   stored credentials back from the API.
4. **Runbooks**: update incident, notification, and ownership documentation so that
   operators use queues, assignees, and saved views consistently.

The upgrade is considered complete when the dashboard documentation, CI gates, and
runtime behavior all describe the same canonical surface. If your runbooks still refer
to removed APIs or deprecated product areas, treat that as unfinished migration work.
